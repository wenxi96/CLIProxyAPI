## Findings

### H-01 Usage v2 核心事实语义尚未裁决

- Evidence: 上游冲突实现会在缺少 total_tokens 时求和，并用字段存在性替代 fork 的 numeric presence；fork 历史契约要求 parser 不合成 reported total，由 internal/usage 集中计算，并区分 explicit zero 与 missing usage。
- Impact: 可能把 computed total 误记为 provider-reported total、双算 reasoning/cache，或把仅有 response tier 的响应误判为 observed-zero usage。
- Recommendation: 明确 parser、UsageObserved、reported/computed total、tier-only metadata 和 cache aliases 的逐字段契约，并保留回归测试。
- Disposition: fixed。

### H-02 32 个自动合并热点没有逐项审查清单

- Evidence: 冲突预检只记录数量；实际 43 个重叠路径还包括 server.go、config/watcher、management、SDK service/conductor 和多个 provider executor。
- Impact: 自动合并可编译但仍可能静默改变配置热更、路由、usage 终态或 provider 行为。
- Recommendation: 增加完整 43-path ledger，逐项记录双方意图、合并策略、保护断言和验证入口。
- Disposition: fixed。

### H-03 SelectAuthByKind 与 scoped pool 存在具体行为风险

- Evidence: 预合并结果中的 SelectAuthByKind 会先调用可能执行 markScopedPoolSelected 的 pickNext，再过滤 auth kind；fork 的 MarkSelected 会更新 LastSelectedAt 并刷新池成员。
- Impact: Alpha Search 跳过 API-key、最终选择 OAuth 时，可能先改变不合格凭证的 scoped-pool 排序或成员状态。
- Recommendation: 在记录 selected 状态前过滤 kind，或证明副作用符合预期；覆盖 mixed API-key/OAuth、plugin scheduler、Home 和 scoped-pool。
- Disposition: fixed。
