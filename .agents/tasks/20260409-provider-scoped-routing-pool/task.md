# 任务说明

## 目标

为认证文件与配置型 AI 供应商凭证增加“按供应商类别独立启用的范围轮询”能力，并保证只有在配置显式开启后才生效；未开启时，必须完全保持当前调度与管理行为不变。

## 范围

- 设计按 provider category 独立建池的范围轮询能力
- 明确认证文件与配置型 provider 凭证的统一运行时建模方式
- 设计显式开关、池大小、额度阈值、连续错误阈值等配置结构
- 设计池内成员选择、踢出、补位、降权与日志方案
- 设计认证文件页、AI Providers 页与配置中心的前端改动方案
- 设计兼容性、回退与测试策略

## 非目标

- 不实现全局单池
- 不在未开启配置时对默认调度链路做任何隐式介入
- 不把不支持额度检查的 provider 强行纳入额度阈值判断
- 不在本轮直接进入代码实现

## 验收

- 范围轮询按供应商类别独立生效，而非全局共享池
- 只有显式开启的 provider 才会进入范围轮询
- 未开启时保持当前 `round-robin` / `fill-first` 与现有管理展示逻辑不变
- 设计覆盖后端配置、运行时调度、管理接口、前端展示与测试策略
- 设计明确说明与现有功能的兼容边界和回退方式
