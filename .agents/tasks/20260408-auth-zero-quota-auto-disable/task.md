# 任务说明

## 目标

为支持真实额度查询的认证文件新增“额度真实耗尽后自动禁用”能力，并在配置中心提供显式开关，避免额度已耗尽的认证文件继续参与运行时请求重试与路由。

## 范围

- 为支持真实额度查询的认证文件提供共享额度检查能力
- 在运行时失败后异步触发单认证文件额度确认任务
- 明确确认额度耗尽后，自动将该认证文件切换为禁用状态并持久化
- 增加配置项 `quota-exceeded.auto-disable-auth-file-on-zero-quota`
- 为管理配置接口补齐读取与更新端点
- 修复各存储后端在 `auth.Storage != nil` 场景下的 `disabled` 元数据落盘一致性
- 补齐相关单元测试

## 非目标

- 不对无法进行真实额度查询的认证文件启用自动禁用
- 不根据“疑似耗尽”或单次运行时错误直接禁用认证文件
- 不在本轮实现自动恢复
- 不扩展管理前端页面
- 不改变现有请求失败后的重试主链路

## 验收

- 仅支持真实额度查询的认证文件会参与自动禁用判断
- 运行时失败后不会阻塞当前请求链路，而是异步触发额度确认
- 同一认证文件在同一时间只会存在一个额度确认任务
- 只有额度查询明确确认剩余额度为 `0` 或等价耗尽结果时才会自动禁用
- 自动禁用结果会正确写回当前存储后端，服务重启后不会回弹
- 配置接口可读写该开关
- 相关测试通过
