# 任务说明

## 目标

补齐当前 `wenxi96/CLIProxyAPI` fork 的两项缺失能力：

1. 让仓库具备可独立分发的 Linux 一键安装 / 更新脚本，不依赖 `cliproxyapi-tool`
2. 让仓库具备可独立发布和引用的 Docker 镜像链路，不再默认指向官方镜像

## 范围

- 规划后端仓库内安装器脚本、自带更新脚本、systemd 启动脚本的落点与实现边界
- 规划 Docker 镜像发布工作流、默认镜像地址与文档改造
- 规划 README 与本地部署文档入口调整

## 非目标

- 不把 `cliproxyapi-tool` 作为仓库内安装依赖
- 不在本任务内迁移整套上游帮助站
- 不在计划阶段直接修改 Docker Hub 凭证或创建外部仓库

## 验收口径

- 形成一份 canonical implementation plan
- 计划覆盖脚本、工作流、文档、验证与风险边界
- 计划可直接作为后续实现依据，不依赖额外口头补充
